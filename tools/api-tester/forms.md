## 使用说明

1. **配置顺序**: 按照项目配置 → API配置 → 测试用例配置 → 断言配置 → 报告配置的顺序进行
2. **API配置**: 确保API URL和认证配置正确
3. **测试用例**: 合理设计测试用例，覆盖主要业务场景
4. **断言设置**: 设置合适的断言规则验证API响应
5. **性能测试**: 根据业务需求配置性能测试参数
6. **安全测试**: 配置安全测试规则，检查API安全性
7. **报告生成**: 选择合适的报告格式和分发方式
