## 使用说明

1. **配置顺序**: 按照项目配置 → 版本控制配置 → 变更分类配置 → 内容配置 → 模板配置 → 输出配置的顺序进行
2. **Git配置**: 确保Git仓库路径正确，分支和标签设置合理
3. **版本范围**: 根据需要设置合适的版本范围和时间范围
4. **变更分类**: 合理配置变更类型和分组规则
5. **内容设置**: 根据项目需求配置标题、描述和条目格式
6. **模板选择**: 选择合适的输出格式和模板
7. **输出设置**: 配置输出路径和发布方式
