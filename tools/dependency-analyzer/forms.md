## 使用说明

1. **配置顺序**: 按照项目配置 → 依赖源配置 → 安全分析配置 → 版本分析配置 → 报告配置 → 集成配置的顺序进行
2. **项目设置**: 根据项目类型选择合适的包管理器和配置文件
3. **安全分析**: 配置漏洞数据库、许可证策略和质量指标
4. **版本管理**: 设置版本兼容性规则和依赖树分析选项
5. **报告生成**: 选择合适的报告格式和分发方式
6. **集成配置**: 根据需要配置CI/CD、IDE和监控集成
