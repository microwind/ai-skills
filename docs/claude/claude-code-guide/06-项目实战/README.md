# 项目实战：完整系统开发指南

通过两个完整的项目，学习如何用 Claude Code 从零构建生产级系统。

---

## 项目概览

### 项目 1：Java 用户登录系统

**技术栈**：Java 17 + Spring Boot + PostgreSQL + React

**包含内容**：
- 完整的后端 API（认证、用户管理）
- React 前端页面
- 数据库设计和初始化脚本
- 完整的测试套件
- API 文档

**学习重点**：
- Java/Spring Boot 最佳实践
- JWT 认证实现
- 数据库设计
- 前后端集成

**预期时间**：2-3 小时

---

### 项目 2：Go 用户登录系统

**技术栈**：Go 1.21 + Gin + PostgreSQL + Docker

**包含内容**：
- 高性能的后端 API
- Docker 快速部署
- PostgreSQL 集成
- 完整的单元和集成测试
- 性能基准测试

**学习重点**：
- Go 并发编程
- Gin 框架最佳实践
- Docker 容器化
- 高性能系统设计

**预期时间**：2-3 小时

## 每个项目的结构

### 文档结构

每个项目包括：

1. **完整指南** - 从零到一的步骤
2. **提示词** - 与 Claude 的对话示例
3. **代码文件** - 所有源代码
4. **配置文件** - 环境配置、依赖管理
5. **启动脚本** - 快速运行项目

### 代码文件夹结构

```
java/code/
├── User.java              # 用户模型
├── AuthService.java       # 认证服务
├── AuthController.java    # API 控制器
├── JwtTokenProvider.java  # JWT 工具
├── LoginPage.tsx          # React 登录页
├── pom.xml               # Maven 依赖
├── application.yml       # Spring 配置
└── schema.sql            # 数据库初始化

go/code/
├── models/user.go        # 用户模型
├── jwt.go               # JWT 工具
├── handlers/auth.go     # API 处理函数
├── main.go              # 应用主程序
├── go.mod               # 依赖管理
└── docker-compose.yml   # Docker 配置
```

## 常见问题

**Q：代码能直接用到生产吗？**
A：大部分代码可以。少数地方需要根据生产环境调整（如数据库连接、API 密钥等）。

**Q：项目之间有什么区别？**
A：都是用户登录系统，但 Java 包含了前端，Go 重点是后端和性能。选择适合你的技术栈。

**Q：需要多长时间完成？**
A：2-3 小时。包括理解代码、运行、测试、学习。

**Q：能不能修改需求？**
A：完全可以！按照相同的工作流和提示词修改即可。
