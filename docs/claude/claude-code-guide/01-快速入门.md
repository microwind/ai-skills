# Claude Code 快速入门：5 分钟了解核心概念

## 什么是 Claude Code？

Claude Code 是 Anthropic 官方推出的 CLI 工具，让你可以通过自然语言与 Claude AI 进行编程对话。它不是代码生成器，而是**编程伙伴**。

### 核心特点

| 特性 | 说明 |
|------|------|
| **对话式** | 用自然语言描述需求，Claude 理解并帮助实现 |
| **上下文感知** | Claude 记住整个项目背景，建议更连贯准确 |
| **多步骤工作流** | 支持复杂的、分步骤的任务自动化 |
| **长期记忆** | 可以恢复之前的会话，继续未完成的工作 |
| **代码理解** | 不仅生成代码，更能解释和优化代码 |

## 怎么开始使用？

### 安装

```bash
# 使用 npm/yarn 安装
npm install -g @anthropic-ai/claude-code

# 或使用 Homebrew（macOS）
brew install claude-code

# 验证安装
claude --version
```

### 第一个命令

```bash
# 最简单的使用
$ claude "输出 'Hello World' 的 Python 代码"

# 结果
print("Hello World")
```

### 三种使用模式

#### 模式 1：快速咨询

```bash
# 问一个问题
$ claude "什么是闭包？用 JavaScript 给个例子"

# Claude 立即回答并给出代码示例
```

#### 模式 2：代码分析

```bash
# 分析某个文件
$ claude --file src/utils.ts "这个文件有性能问题吗？"

# Claude 分析后给出建议
```

#### 模式 3：多步骤工作流

```bash
# 执行多个步骤
$ claude "
1. 分析需求
2. 设计架构
3. 实现代码
4. 编写测试
"

# Claude 依次完成每个步骤
```

## 核心概念

### 概念 1：对话记忆

Claude Code 会记住你的整个对话历史。

```bash
# 第 1 条：介绍项目
$ claude "我在用 React 构建一个电商平台"

# 第 2 条：Claude 已记住背景
$ "现在加一个购物车功能"
# Claude 知道你说的是什么，生成的代码会与 React 项目一致

# 第 3 条：继续深化
$ "用 Redux 管理购物车状态"
# Claude 知道项目的每个细节
```

**好处**：你不需要每次都重复背景信息，可以专注于当前任务。

### 概念 2：上下文感知

Claude 理解代码的上下文，给出更精准的建议。

```bash
# 不好的提问（没有上下文）
$ claude "如何优化这个函数的性能？"

# 好的提问（有上下文）
$ claude "src/api/handler.ts:45 的函数性能太差，怎么优化？"
# Claude 知道具体位置，可以针对性地优化
```

### 概念 3：工作流驱动

复杂任务分解成多个可验证的步骤。

```bash
# 一次性要求（容易出错）
$ claude "写一个完整的用户登录系统"

# 工作流方式（更稳健）
$ claude "
步骤 1：设计数据库 (User, RefreshToken 表) → 输出：DDL 脚本
步骤 2：写后端服务 (AuthService) → 输入：数据库设计 → 输出：Java 代码
步骤 3：写 API 控制器 → 输入：服务代码 → 输出：REST API
步骤 4：写单元测试 → 验证前面的步骤
步骤 5：写前端页面 → 验证整个流程
"
```

## 常用工作流模板

### 工作流 1：快速原型化

```
需求 → 代码生成 → 验证 → 完善
```

**时间**：30 分钟
**用途**：快速验证想法

```bash
$ claude "
用 React 写一个 Todo 应用前端：
1. 输入框和 Add 按钮
2. Todo 列表
3. 删除功能
4. 响应式设计
"
```

### 工作流 2：完整项目开发

```
需求分析 → 架构设计 → 后端实现 → 前端实现 → 测试 → 文档
```

**时间**：1-2 天
**用途**：构建生产级系统

```bash
$ claude "
构建用户登录系统（Java + React）：
步骤 1：数据库设计 (DDL)
步骤 2：后端 API (Spring Boot)
步骤 3：前端页面 (React)
步骤 4：集成测试
步骤 5：API 文档 (Swagger)
"
```

### 工作流 3：问题诊断

```
现象描述 → 日志分析 → 根因定位 → 修复方案 → 测试
```

**时间**：15-30 分钟
**用途**：快速 Debug

```bash
$ claude "
API 返回 500 错误，日志是：
[日志内容]

1. 分析可能的原因
2. 定位具体问题行
3. 给出修复代码
4. 写单元测试防止回归
"
```

## 实际对比：传统 vs Claude Code

### 案例 1：写一个排序算法

**传统方式**
```
打开编辑器 → 手写代码 → 测试 → Debug → 优化 → 文档
时间：60 分钟
```

**Claude Code 方式**
```bash
$ claude "
用 TypeScript 实现快速排序：
1. 基础实现
2. 性能优化（三路分割）
3. 单元测试
4. 时间复杂度分析
"
结果：完整的代码 + 测试 + 分析
时间：5 分钟
```

### 案例 2：修复登录 Bug

**传统方式**
```
查看错误日志 → 追踪堆栈 → 理解逻辑 → 找到问题 → 修复 → 测试
时间：45 分钟
```

**Claude Code 方式**
```bash
$ claude "登录后页面闪屏，[上传相关文件]，是什么原因？"
结果：立即定位问题、给出修复代码、写测试
时间：5 分钟
```
