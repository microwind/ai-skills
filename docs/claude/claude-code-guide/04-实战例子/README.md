## 例子的共同模式

所有例子都遵循同样的模式，这也是最有效的 Claude Code 使用方式：

### 模式：对话 → 验证 → 迭代 → 文档

```
1. 对话：清晰地表达需求
   ↓
2. 验证：检查生成的代码是否符合需求
   ↓
3. 迭代：提出改进建议，继续对话
   ↓
4. 文档：生成或完善文档
```

**示例**：

```bash
# 第 1 步：对话
$ claude "写快速排序算法"
# Claude 生成基础实现

# 第 2 步：验证
# 你运行代码，检查是否正确

# 第 3 步：迭代
$ "现在优化到 O(n log n)"
# Claude 优化代码

# 第 4 步：文档
$ "解释一下三路分割的优势"
# Claude 详细讲解
```

## 关键技巧速查

以下是这 5 个例子中用到的关键技巧：

| 技巧 | 出现在 | 说明 |
|------|--------|------|
| 上下文感知 | 全部 | 多轮对话不重复说明背景 |
| 代码块标记 | 全部 | 用\`\`\`语言 标记代码 |
| 文件定位 | Bug 定位、测试 | 精准指向要改的代码位置 |
| 渐进式精化 | 排序、页面、测试 | 逐步优化而非一次性完成 |
| 后台任务 | 测试 | 同时运行测试和编码 |
| 对比式提问 | 响应式页面、测试 | 对比多个方案选择最好的 |
| 约束条件 | 全部 | 明确要求让生成代码更优 |

## 实战建议

### 💡 学习建议

1. **边学边做**：不要只看，要真的与 Claude Code 互动
2. **修改和实验**：修改提示词，看看 Claude 会给出什么不同的答案
3. **关注细节**：注意 Claude 如何思考和解决问题
4. **记录模式**：每个例子都有特定的解题模式，记录下来

### 🚀 实践建议

1. **复用提示词**：这些提示词可以直接用于你的真实项目
2. **扩展代码**：把例子中的代码扩展到实际需求
3. **组合技巧**：同时使用多个技巧解决更复杂的问题

### ⚡ 时间管理建议

- 每个例子预留 15-20 分钟
- 第一遍学习：理解思路
- 第二遍学习：自己重新做一遍
- 第三遍：应用到实际项目
