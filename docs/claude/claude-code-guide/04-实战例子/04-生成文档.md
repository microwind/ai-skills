# 例子 4：生成高质量设计文档

**场景**：新开发者加入团队，需要快速理解认证系统。用 Claude Code 自动生成完整的设计文档。

**学习目标**：
- 学习文档自动化
- 理解系统设计文档结构
- 掌握架构可视化

---

## 背景

你有一个完整的认证系统（多个文件、复杂逻辑），需要文档。手动写文档：2-3 小时。用 Claude Code：15 分钟。

---

## 第一步：收集信息

### 提示词

```bash
$ claude "
基于这些文件生成一份完整的认证系统设计文档：
- src/auth/middleware.ts（认证中间件）
- src/auth/jwt.ts（JWT 处理）
- src/auth/service.ts（认证服务）
- src/models/User.ts（用户模型）
- src/controllers/AuthController.ts（API 控制器）

文档应包括：
1. 系统架构图（Mermaid）
2. 认证流程（序列图）
3. 关键数据结构
4. API 端点列表
5. 安全注意事项
6. 集成指南
"
```

---

## 第二步：生成架构图

输出示例：

```markdown
## 系统架构

\`\`\`mermaid
graph TB
  Client[客户端]
  API[API 网关]
  Auth[认证服务]
  DB[(数据库)]
  Cache[(缓存)]

  Client -->|登录| API
  API -->|验证| Auth
  Auth -->|查询| DB
  Auth -->|缓存| Cache
  Auth -->|生成 Token| Client
\`\`\`
```

---

## 第三步：生成流程图

```markdown
## 认证流程

\`\`\`mermaid
sequenceDiagram
  User->>Client: 输入凭证
  Client->>API: POST /login
  API->>Auth: validateCredentials()
  Auth->>DB: 查询用户
  DB-->>Auth: 返回用户信息
  Auth->>Auth: 验证密码
  Auth->>Cache: 缓存 JWT
  Auth-->>API: 返回 token
  API-->>Client: 成功响应
  Client->>Client: 保存 token
\`\`\`
```

---

## 第四步：API 文档

### 提示词

```bash
$ claude "
基于代码生成 OpenAPI/Swagger 格式的 API 文档，包括：
1. 所有端点（GET, POST, PUT, DELETE）
2. 请求参数和响应格式
3. 错误码说明
4. 认证要求
5. 速率限制

输出 YAML 格式
"
```

---

## 第五步：最佳实践和安全

### 提示词

```bash
$ claude "
补充文档的安全部分：
1. JWT token 验证流程
2. 常见安全陷阱和预防
3. CORS 配置
4. CSRF 保护
5. 速率限制配置

用表格格式
"
```

---

## 第六步：集成指南

### 提示词

```bash
$ claude "
为新开发者写集成指南：
1. 如何在我的应用中使用这个认证系统？
2. 代码示例（Express, NestJS, Django）
3. 环境变量配置
4. 常见问题 FAQ

让非核心开发者也能快速上手
"
```

---

```
设计文档

│   ├── 架构图

├── 安全考虑

│   └── 配置说明
└── FAQ
```
