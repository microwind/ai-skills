# Claude Code 常用技巧：11 个高效编程技巧

学完这 11 个技巧，你可以把代码编写效率提升 3 倍。

---

## 技巧 1：善用「上下文感知」进行多轮对话

**核心**：Claude Code 记住整个对话历史，你不需要重复背景信息。

```bash
# 第一次提问
$ claude "分析这个 React 组件的性能问题"

# 后续提问，Claude 已知道背景
$ "现在用 memo 优化它"
# Claude 知道你说的是哪个组件

$ "再加上 useCallback"
# 基于前面的优化继续
```

**为什么有效**：避免重复解释背景，每次都能基于前面的结果继续优化。

**实战场景**：
- 调试复杂 Bug 时逐步深入
- 优化代码时逐步增加约束条件
- 学习新框架时逐个提问

## 技巧 2：利用 `/tasks` 命令管理后台任务

Claude Code 支持在后台运行长时任务，不阻塞你的命令行。

```bash
# 启动一个耗时的构建任务
$ claude --background "构建整个项目并运行所有测试"

# 继续其他工作...
$ # 同时编写新功能

# 查看所有后台任务
$ /tasks

# 获取某个任务的结果
$ /tasks <task-id>
```

**何时使用**：
- 运行测试套件（通常需要几分钟）
- 构建大型项目
- 生成覆盖率报告
- 运行代码分析工具

**效果**：充分利用系统资源，编写代码的同时运行测试。

## 技巧 3：精准使用「文件定位」减少噪音

告诉 Claude 要修改哪个文件，而不是让它搜索整个项目。

```bash
# ❌ 不好（Claude 需要搜索整个项目）
$ claude "登录验证逻辑有问题，修复一下"

# ✅ 好（直接定位到文件）
$ claude "src/auth/login.ts 的验证逻辑有问题，修复一下"

# ✅ 更好（定位到具体行）
$ claude "src/auth/login.ts:45 这里的验证逻辑不对，应该..."
```

**为什么有效**：
- Claude 不需要理解无关代码
- 更快速地找到问题
- 修复建议更精准

**格式**：`file_path:line_number`

## 技巧 4：使用 Markdown 代码块指定编程语言

清晰的语言标记让 Claude 提供准确的建议。

```bash
$ claude "
我的代码有问题：

\`\`\`typescript
interface User {
  id: number;
  name: string;
}

const users: User[] = [];
users.push({ id: 1 }); // 这里报错了
\`\`\`

为什么会报错？
"
```

Claude 会立即识别是 TypeScript 代码，告诉你：

> 因为 User 接口要求有 `name` 字段，你没有提供。

**好处**：
- 语法高亮提醒 Claude
- 语言特定的最佳实践建议
- 错误识别更准确

**支持的语言**：js, ts, jsx, tsx, java, go, py, rust, sql, yaml, json 等

## 技巧 5：利用「分支思维」处理复杂决策

遇到多个实现方案时，先让 Claude 列出选项，再深入某个方案。

```bash
# 第一步：列出方案
$ claude "
在我的 Node.js 项目中实现缓存，有什么方案？
请列出各自的优缺点和适用场景。
"

# Claude 列出：Redis、内存缓存、数据库缓存等

# 第二步：选择并深入
$ "我选择 Redis，具体怎么集成？"

# 第三步：具体实现
$ "写个 Node.js + Redis 的缓存工具类"
```

**为什么有效**：
- 避免一开始就陷入实现细节
- 能看到全局
- 做出更好的技术决策

## 技巧 6：使用 `--resume` 恢复之前的会话

处理跨越多天的大型项目时，恢复之前的会话。

```bash
# 查看会话 ID
$ /sessions

# 恢复某个会话（可能是昨天的）
$ claude --resume <session-id> "继续实现登录功能"

# Claude 知道整个项目背景，建议更连贯
```

**场景**：
- 多天项目开发
- 项目暂停后恢复
- 跨越多个工作周期

## 技巧 7：让 Claude 生成「测试优先」的代码

先要求 Claude 写测试，再写实现代码。

```bash
# 第一步：写测试
$ claude "
为用户注册功能写单元测试：
- 成功注册
- 邮箱已存在
- 密码过弱
- 邮箱格式错误

使用 Jest 框架
"

# 第二步：实现代码
$ "现在基于这些测试写实现代码"

# 结果：实现代码完全符合测试需求
```

**好处**：
- 确保代码质量
- 测试即文档
- 实现代码更稳定

## 技巧 8：精用「对比式」提问找出最佳实践

```bash
$ claude "
对比以下两种错误处理方式的优缺点：

1. try-catch 包装一切
\`\`\`typescript
try {
  const result = await api.call();
} catch (e) {
  console.error(e);
}
\`\`\`

2. Result<T, E> 类型模式
\`\`\`rust
match result {
  Ok(value) => { /* handle */ },
  Err(error) => { /* handle */ },
}
\`\`\`

在我的 TypeScript 项目中哪个更好？
"
```

Claude 会结合你的项目特点给出建议。

## 技巧 9：利用代码片段快速定位

使用 `file_path:line_number` 格式快速指向代码位置。

```bash
# 定位具体行数
$ claude "src/api/handler.ts:45 这里的错误处理有问题"

# Claude 立即理解是哪一行，建议更精准
```

**格式示例**：
- `src/main.ts:10` - 第 10 行
- `src/config/index.ts:5-15` - 第 5-15 行
- `src/utils/helper.ts:calculateTotal` - 找到函数然后在那行

## 技巧 10：使用「渐进式精化」优化代码

不要一次性要求「优化整个函数」，而是逐步改进。

```bash
# 第一步：分析
$ claude "这个函数的时间复杂度是多少？"

# Claude 分析：O(n²)

# 第二步：优化
$ "怎样优化到 O(n)？"

# Claude 给出优化方案

# 第三步：完善
$ "现在加上错误处理"

# 第四步：测试
$ "写个性能测试确保 O(n) 复杂度"
```

**为什么有效**：
- 避免优化引入新 Bug
- 每步都可验证
- 最终代码质量高

---

## 技巧 11：善用「约束条件」获得更好的代码

明确指定约束，让 Claude 生成最优解决方案。

```bash
# 不好（太宽泛）
$ claude "写一个 JSON 解析函数"

# 好（有约束）
$ claude "
写一个 JSON 解析函数，要求：
- 必须处理格式错误
- 性能要超过 JSON.parse 的 80%
- 代码不超过 50 行
- 支持注释（// 和 /* */）
- 返回类型是 Result<T, Error>
"
```

**约束条件的作用**：
- 指导 Claude 选择最优实现
- 明确非功能需求
- 生成的代码更实用

## 实战组合技巧

### 组合 1：快速 Debug（技巧 3 + 9 + 10）

```bash
$ claude "
src/api/handler.ts:45 这个函数返回错误的结果
1. 分析问题原因
2. 给出修复代码
3. 写单元测试防止回归
"
```

### 组合 2：高质量功能开发（技巧 7 + 10 + 11）

```bash
$ claude "
为用户认证写完整实现，要求：
1. 先写测试（覆盖率 >90%）
2. 再写实现代码
3. 性能要支持 1000 QPS
4. 必须处理所有异常情况
"
```

### 组合 3：长期项目管理（技巧 6 + 2 + 5）

```bash
# 第一天
$ claude "设计电商系统架构" # 创建会话

# 第二天
$ claude --resume <session-id> "现在实现支付模块" # 恢复会话
$ claude --background "运行完整测试" # 后台任务

# 第三天
$ claude --resume <session-id> "有 3 个实现方案，对比它们" # 分支思维
```
