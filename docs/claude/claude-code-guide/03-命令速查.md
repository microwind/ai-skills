# Claude Code 命令速查

快速查找和使用 Claude Code 的所有主要命令。

---

## 核心命令

### 基础对话

```bash
# 简单提问
$ claude "输出 'Hello World'"

# 长提问（多行）
$ claude "
写一个快速排序函数：
- 使用 TypeScript
- 包括类型定义
- 写单元测试
"

# 从文件读取提问（提示词太长时）
$ claude < prompt.txt
```

**参数**：无额外参数

---

### 分析文件

```bash
# 分析单个文件
$ claude --file src/utils.ts "这个文件有性能问题吗？"

# 分析多个文件
$ claude --file src/api.ts --file src/db.ts "这两个文件如何集成？"

# 分析目录（会扫描所有文件）
$ claude --file src/ "项目的代码结构如何？"
```

**常见用途**：
- 代码审查
- 性能分析
- 理解复杂模块
- 检查安全漏洞

---

### 后台运行任务

```bash
# 启动后台任务，不阻塞命令行
$ claude --background "运行完整测试套件"

# 查看所有后台任务
$ /tasks

# 查看某个任务的详细状态
$ /tasks <task-id>

# 获取任务的最终结果
$ /tasks <task-id> --output
```

**适用场景**：
- 运行测试（通常 5-10 分钟）
- 构建项目（通常 2-5 分钟）
- 生成报告
- 代码分析

**提示**：在后台任务运行期间，你可以继续其他工作。

---

### 恢复会话

```bash
# 查看所有会话
$ /sessions

# 恢复某个会话
$ claude --resume <session-id> "继续之前的工作"

# 查看会话信息
$ /sessions <session-id>
```

**会话适用于**：
- 多天项目开发
- 项目暂停后恢复
- 保持项目背景记忆

## 高级参数

### 模式参数

```bash
# 交互模式（继续对话）
$ claude
> 问题 1
Claude 回答...
> 问题 2（会记住问题 1 的背景）
Claude 回答...
> exit 或 quit 退出

# 单次模式（只回答一个问题）
$ claude "一个问题"
```

---

### 输出参数

```bash
# 以 JSON 格式输出
$ claude --format json "给我一个 JSON 结构"

# 保存输出到文件
$ claude "生成测试文件" > tests.ts

# 查看更详细的输出（调试时有用）
$ claude --verbose "做某个任务"
```

## 工作流命令

### 多步骤工作流

```bash
$ claude "
步骤 1：设计数据库
步骤 2：实现后端
步骤 3：写测试
步骤 4：生成文档
"

# Claude 会依次完成每个步骤
```

### 验证工作流

```bash
$ claude "
执行以下工作流，每步后验证：

步骤 1：编写代码 → 输出：main.py
验证：代码能否导入？

步骤 2：运行测试 → 输入：main.py
验证：测试是否全部通过？

步骤 3：检查覆盖率 → 验证：覆盖率 > 80%？
"
```

## 常用命令组合

### 组合 1：快速代码审查

```bash
$ claude --file src/critical.ts "
检查这个文件：
1. 是否有性能问题？
2. 是否有安全漏洞？
3. 代码风格是否一致？
4. 给出改进建议
"
```

### 组合 2：Bug 诊断工作流

```bash
# 第 1 步：分析症状
$ claude "这个 API 返回 500 错误，可能是什么原因？"

# 第 2 步：查看日志
$ claude --file logs.txt "根据这些日志找出真实原因"

# 第 3 步：查看代码
$ claude --file src/api.ts:45-60 "这里的逻辑有问题吗？"

# 第 4 步：修复和测试
$ claude "修复这个问题，并写单元测试"

# 第 5 步：后台验证
$ claude --background "运行完整测试确保没有回归"
```

### 组合 3：完整项目开发

```bash
# 第 1 次会话：设计
$ claude "设计电商系统的数据库和 API"
# 记下 <session-id>

# 第 2 天：实现后端
$ claude --resume <session-id> "现在实现后端 API"

# 第 3 天：实现前端
$ claude --resume <session-id> "写 React 前端"

# 第 4 天：测试和文档
$ claude --resume <session-id> "写测试和 API 文档"
$ claude --background "运行完整测试"
```

## 调试和帮助

### 获取帮助

```bash
# 查看所有可用命令
$ claude --help

# 查看特定命令的帮助
$ claude --help <command>

# 查看版本信息
$ claude --version

# 查看配置信息
$ claude --config
```

---

### 调试模式

```bash
# 启用详细输出，看 Claude 的思考过程
$ claude --verbose "做某个任务"

# 查看 API 调用情况（高级用户）
$ claude --debug "任务"

# 显示完整的错误堆栈
$ claude --stack-trace "任务"
```

## 最常用的 5 个命令

按使用频率：

### 1️⃣ 快速提问
```bash
$ claude "你的问题"
```
用途：日常编程问题

### 2️⃣ 分析文件
```bash
$ claude --file <path> "分析内容"
```
用途：代码审查、性能分析

### 3️⃣ 后台任务
```bash
$ claude --background "耗时任务"
```
用途：测试、构建

### 4️⃣ 恢复会话
```bash
$ claude --resume <id> "继续工作"
```
用途：多天项目

### 5️⃣ 交互模式
```bash
$ claude
```
用途：连续对话、迭代开发

## 键盘快捷键（交互模式）

在 `claude` 交互模式中：

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+C` | 中断当前回复 |
| `↑` / `↓` | 浏览历史命令 |
| `Ctrl+L` | 清屏 |
| `Ctrl+D` 或 `exit` | 退出交互模式 |
| `Tab` | 自动完成（如果支持） |

## 环境变量

如果你经常设置某些参数，可以用环境变量：

```bash
# 设置默认输出格式
export CLAUDE_FORMAT=json

# 设置 API 密钥
export ANTHROPIC_API_KEY=your_key_here

# 验证配置
$ claude --config
```
