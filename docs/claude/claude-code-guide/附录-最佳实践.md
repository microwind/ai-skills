# 附录：Claude Code 最佳实践

总结 Claude Code 的企业级使用规范和常见陷阱。

---

## 第一部分：提示词最佳实践

### 原则 1：明确的目标

```bash
❌ 不好
$ claude "写一个排序算法"

✅ 好
$ claude "
用 TypeScript 实现快速排序：
- 包括类型定义
- 处理边界情况
- 添加单元测试
- 时间复杂度 O(n log n)
"
```

### 原则 2：充分的上下文

```bash
❌ 不好
$ claude "这里有 Bug，修复一下"

✅ 好
$ claude "
src/api/handler.ts:45 的登录逻辑有 Bug
症状：用户登录后页面闪屏
代码片段：
\`\`\`typescript
// 代码
\`\`\`
问题分析和修复建议
"
```

### 原则 3：明确的约束条件

```bash
❌ 太宽泛
$ claude "写一个缓存系统"

✅ 明确约束
$ claude "
写一个 Redis 缓存系统：
- 支持 get/set/delete 操作
- 自动过期时间设置
- 性能支持 100k QPS
- 代码不超过 500 行
- 使用 Node.js + redis 库
"
```

### 原则 4：循序渐进

```bash
❌ 一次性要求
$ claude "完整的用户认证系统"

✅ 循序渐进
$ claude "
步骤 1：数据库设计 (User, RefreshToken 表)
步骤 2：JWT 工具类
步骤 3：注册/登录 API
步骤 4：单元测试
步骤 5：集成测试
"
```

## 第二部分：常见陷阱和解决方案

### 陷阱 1：过度依赖 AI

**症状**：完全相信 Claude 生成的代码，不进行验证。

**后果**：
- 代码可能有逻辑错误
- 安全漏洞被忽视
- 性能问题未发现

**解决方案**：
```bash
# 始终验证生成的代码
1. 阅读代码逻辑
2. 运行测试
3. 进行代码审查
4. 在测试环境验证
```

### 陷阱 2：提示词过于简洁

**症状**：只说需要什么，不说怎么做。

```bash
❌ 不好
$ claude "写登录功能"

✅ 好
$ claude "
写登录 API：
- 接收邮箱和密码
- 验证邮箱存在
- 验证密码正确
- 返回 JWT token
- 处理所有异常情况
"
```

**后果**：生成的代码质量不稳定。

### 陷阱 3：忽视上下文限制

**症状**：信息太多，超过 Claude 的处理能力。

```bash
❌ 不好
# 一次上传整个项目的所有代码
$ claude --file src/ "分析这个项目"

✅ 好
# 分开分析关键模块
$ claude --file src/auth/ "认证模块有性能问题吗？"
$ claude --file src/api/ "API 设计合理吗？"
```

**解决方案**：分块上传，逐个分析。

### 陷阱 4：工作流步骤不清楚

**症状**：步骤之间的依赖关系不明确。

```bash
❌ 不好
$ claude "
做 A
做 B
做 C
"

✅ 好
$ claude "
步骤 1：A（无依赖）→ 输出：文件 1
步骤 2：B（需要文件 1）→ 输出：文件 2
步骤 3：C（需要文件 2）→ 输出：文件 3
"
```

### 陷阱 5：安全问题被忽视

**症状**：Claude 生成的代码有安全漏洞。

常见的安全问题：
- SQL 注入
- XSS 攻击
- CSRF 漏洞
- 硬编码的密钥
- 不安全的密码存储

**解决方案**：
```bash
$ claude "
检查这段代码的安全问题：
\`\`\`
代码
\`\`\`

特别关注：
- SQL 注入
- XSS
- 认证和授权
- 敏感数据处理
"
```

### 代码质量检查清单

在使用 Claude 生成的代码前，检查：

- [ ] 代码能编译/运行吗？
- [ ] 所有测试都通过吗？
- [ ] 代码风格一致吗？
- [ ] 有没有代码坏味道？
- [ ] 性能满足要求吗？
- [ ] 有没有安全漏洞？

- [ ] 错误处理完善吗？
- [ ] 日志足够吗？
- [ ] 可测试性好吗？

```bash

$ prettier --write "**/*.ts"

$ tsc --noEmit

# 单元测试
$ npm test -- --coverage

$ npm audit

$ lighthouse https://yourapp.com
```

## 第六部分：性能优化建议

### 提示词优化

```bash
❌ 低效
$ claude "优化这个函数"

✅ 高效
$ claude "
优化这个函数（当前时间复杂度：O(n²)）：
- 目标复杂度：O(n log n)
- 不能增加内存（O(1) 空间）
- 代码要保持可读性
- 提供性能对比
"
```

### 会话复用

```bash
# 启动一个长期会话
$ claude
> 问题 1
> 问题 2（基于问题 1）
> 问题 3（基于问题 1、2）
# 避免重复上下文加载

# 而不是
$ claude "问题 1"
$ claude "问题 2"（丧失上下文）
$ claude "问题 3"（完全重新开始）
```

## 第七部分：常见问题解答

### Q: Claude Code 生成的代码可以直接用于生产吗？

**A**: 大部分代码质量不错，但需要：
1. 代码审查
2. 完整的测试
3. 安全审查
4. 性能验证
5. 生产环境验证

### Q: 如何处理 Claude 给出错误答案的情况？

**A**:
1. 提供更多上下文
2. 用不同的方式重新提问
3. 指出错误，让 Claude 修正
4. 恢复会话，继续修复

```bash
$ claude "你的前一个答案有问题，原因是..."
```

### Q: 如何减少 Claude 的幻觉？

**A**:
1. 给出具体的约束条件
2. 要求代码尽量简单
3. 要求详细的解释
4. 提供充分的上下文
5. 验证所有输出

### Q: 能否让 Claude 生成特定编程语言的代码？

**A**: 可以，明确指定：

```bash
$ claude "
用 Go 实现...
使用标准库，不依赖第三方库
"
```

### Q: 如何处理长期项目中的上下文丢失？

**A**: 使用恢复会话功能：

```bash
$ claude --resume <session-id> "继续..."
```

## 第八部分：伦理和责任

### 使用 AI 工具的责任

1. **代码所有权**
   - 审查生成的代码
   - 理解代码逻辑
   - 为代码质量负责

2. **安全和隐私**
   - 不上传敏感信息
   - 不上传用户数据

3. **学习和成长**
   - 不要完全依赖 AI
   - 继续学习基础知识
   - 理解设计原理

4. **透明度**
   - 告知团队你用了 AI
   - 文档中说明 AI 辅助
   - 共享实际效果
