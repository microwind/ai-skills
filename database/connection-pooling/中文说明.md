# 数据库连接池管理

## 目的
学习如何高效地管理数据库连接，提升应用性能和稳定性。

## 核心概念

### 为什么需要连接池？
- 创建数据库连接开销大
- 连接复用提升性能
- 防止连接泄漏
- 资源统一管理

### 连接池工作原理
```
1. 初始化 - 创建最小连接数
2. 获取 - 应用从池中获取连接
3. 使用 - 执行数据库操作
4. 归还 - 连接返回池中
5. 销毁 - 定期清理过期连接
```

## 实现方案

### Python - SQLAlchemy示例
```python
from sqlalchemy import create_engine
from sqlalchemy.pool import QueuePool

# 创建连接池
engine = create_engine(
    'postgresql://user:password@localhost/dbname',
    poolclass=QueuePool,
    pool_size=10,           # 活跃连接数
    max_overflow=20,        # 允许超出pool_size的连接数
    pool_recycle=3600,      # 连接回收时间（秒）
    echo=False              # 打印SQL语句
)

# 使用连接
with engine.connect() as conn:
    result = conn.execute("SELECT * FROM users")
```

### Node.js - MySQL Pool示例
```javascript
const mysql = require('mysql2/promise');

const pool = mysql.createPool({
  host: 'localhost',
  user: 'root',
  password: 'password',
  database: 'mydb',
  waitForConnections: true,
  connectionLimit: 10,
  queueLimit: 0
});

// 使用连接
async function query(sql) {
  const connection = await pool.getConnection();
  try {
    const [rows] = await connection.execute(sql);
    return rows;
  } finally {
    connection.release();
  }
}
```

### Go - database/sql示例
```go
import "database/sql"

db, err := sql.Open("mysql", "user:password@/dbname")
defer db.Close()

// 配置连接池
db.SetMaxOpenConns(25)
db.SetMaxIdleConns(5)
db.SetConnMaxLifetime(5 * time.Minute)

// 使用
rows, err := db.Query("SELECT * FROM users")
```

## 关键参数

| 参数 | 说明 | 建议值 |
|------|------|--------|
| pool_size | 核心连接数 | CPU核心数 * 2 |
| max_overflow | 最大溢出连接 | pool_size / 2 |
| max_idle | 最大空闲连接 | 核心连接数的50% |
| timeout | 获取连接超时 | 30秒 |
| recycle | 连接回收时间 | 数据库max_lifetime - 10分钟 |

## 最佳实践

1. **正确关闭连接** - 使用with/try-finally确保连接归还
2. **监控连接状态** - 监控活跃/空闲/等待连接数
3. **设置合理的大小** - 过大浪费资源，过小导致性能下降
4. **处理连接泄漏** - 定期审查日志和监控
5. **使用连接超时** - 防止永久阻塞

## 性能优化

- 连接数 = (核心数 × 2) + 磁盘自旋数
- 对于I/O密集应用可以增加比例
- 定期监控连接使用情况
