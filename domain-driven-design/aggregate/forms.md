# 聚合 - 诊断与规划表

## 识别聚合

### 聚合根识别

```
Order（聚合根）
  ├── OrderLine（从属）
  ├── OrderStatus（值对象）
  └── Money（值对象）

Product（聚合根）
  ├── Price（值对象）
  ├── Description（值对象）
  └── Inventory（值对象）
```

## 边界设计

```
原则：
□ 聚合根是访问点
□ 聚合内通过根修改
□ 外部只引用根
□ 根保证一致性
```

## 检查表

```
□ 有明确的聚合根
□ 定义了聚合边界
□ 根保护内部对象
□ 无循环引用
□ 事务以根为单位
```

聚合是保证数据一致的关键设计。
