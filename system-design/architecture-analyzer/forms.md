## 使用说明

1. **配置顺序**: 按照项目基本信息 → 分析目标配置 → 依赖关系分析 → 设计模式分析 → 质量评估配置 → 安全架构分析 → 输出配置 → 集成配置的顺序进行
2. **分析范围**: 根据项目规模和复杂度选择合适的分析范围和深度
3. **质量标准**: 根据项目要求和团队标准配置质量评估指标
4. **工具选择**: 根据技术栈和现有工具链选择合适的集成工具
5. **结果应用**: 根据分析结果制定改进计划并逐步实施
6. **持续监控**: 定期进行架构分析以监控架构质量变化
