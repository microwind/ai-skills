# 混合推荐系统 - 快速参考指南

## 📋 系统概览

```
电商个性化推荐系统
├─ 规模: 100万商品 × 1000万用户
├─ 性能: <100ms 响应时间
├─ 准确率: >82% (满足 >80% 要求)
├─ 多样性: >5 品类覆盖
└─ 点击率: 15% (相比热门8% 提升87%)
```

## 🏗️ 核心架构 (三层设计)

### 第一层: 召回 (Recall) - 快速筛选
```
目标: 从100万商品 → 500个候选

通道1: 协同过滤 (50%)
  - 原理: 用户行为相似 → 推荐相同商品
  - 性能: <50ms (预计算)
  - 准确率: 85-90%
  - 缺点: 冷启动差

通道2: 内容推荐 (30%)
  - 原理: 基于商品属性和用户偏好
  - 性能: <30ms (预计算)
  - 准确率: 60-70%
  - 优点: 冷启动友好

通道3: 热度排序 (20%)
  - 原理: 推荐热门商品
  - 性能: <20ms
  - 准确率: 基础保障
  - 用途: 兜底方案
```

### 第二层: 排序 (Ranking) - 精细排序
```
目标: 从500候选 → 20个推荐

步骤1: 多样性排序
  - 品类约束: 每类最多3个
  - 冷门商品: 10% 比例混入
  - 目标: 覆盖 5+ 品类

步骤2: CTR预测排序
  - 模型: LightGBM
  - 特征: 用户、商品、交叉、上下文
  - 目标: 优先推荐最可能点击的商品
  - AUC: 0.75-0.80
```

### 第三层: 缓存 (Cache) - 极速响应
```
L1缓存 (热点用户, 1小时)
  - 命中率: 50%
  - 内存: 25GB (500万热点用户)
  - 返回: 秒级

L2缓存 (协同过滤, 24小时)
  - 命中率: 40%
  - 内存: 50GB (全量1000万用户)
  - 返回: <50ms

L3缓存 (相似度, 7天)
  - 命中率: 85%
  - 内存: 20GB (全量100万商品)
  - 返回: <30ms

总缓存: 100GB
硬件: 6台服务器 (3主+3从, Redis Cluster)
命中率: 综合 >80%
```

## 💾 离线计算 (每天 02:00-04:30)

```
1. 协同过滤 (30分钟)
   - 100万商品 × 1000万用户
   - Item-based CF
   - 输出: 相似度矩阵 (20GB)
   - 写入缓存和DB

2. 内容特征 (20分钟)
   - 特征提取 (类别、品牌、价格等)
   - 用户偏好向量
   - 输出: 特征矩阵

3. 用户聚类 & 热度 (40分钟)
   - 用户行为聚类
   - 商品热度统计
   - 冷启动处理

4. CTR模型训练 (50分钟)
   - LightGBM训练
   - 特征工程
   - 模型部署

总耗时: ~2.5小时
```

## ❌ 常见陷阱

### 避免以下问题:

1. **过度优化**
   - ❌ 为了1ms优化增加100倍复杂度
   - ✅ 在证明瓶颈后再优化

2. **忽视冷启动**
   - ❌ 只用CF，新用户无推荐
   - ✅ 用内容推荐 + 热度排序兜底

3. **热门商品堆积**
   - ❌ 一直推荐热门的
   - ✅ 品类约束 + 冷门商品混入

4. **缺少多样性**
   - ❌ 所有推荐都相似
   - ✅ 不同算法融合 + 品类约束

5. **实时更新过频**
   - ❌ 每秒更新所有数据
   - ✅ 离线计算每天1次，在线快速查询
