## 使用说明

1. **配置顺序**: 按照项目基本信息 → Express应用配置 → 路由设计配置 → 中间件配置 → 数据库集成 → 认证和授权 → 错误处理 → 性能优化 → 安全配置 → 测试配置 → 部署配置 → 监控和运维的顺序进行
2. **路由设计**: 根据API需求设计合理的路由结构和RESTful接口
3. **中间件配置**: 按照执行顺序配置中间件，确保安全性和性能
4. **数据库集成**: 选择合适的数据库和ORM，配置连接池和缓存
5. **安全配置**: 始终优先考虑安全配置，特别是生产环境
6. **测试验证**: 在部署前进行充分的测试验证
