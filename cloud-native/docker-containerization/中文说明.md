# Docker容器化

## 目的
学习使用Docker将应用容器化，提升开发和部署效率。

## 核心概念

### Docker工作原理
```
Docker镜像 → Docker容器 → 运行中的进程
  (静态)      (动态)        (执行)
```

### 关键概念
- **镜像（Image）** - 应用的完整打包（代码、依赖、环境）
- **容器（Container）** - 镜像运行的实例
- **仓库（Registry）** - 存储镜像的服务（Docker Hub等）
- **Dockerfile** - 定义如何构建镜像的文件

## Dockerfile编写

### 最佳实践示例
```dockerfile
# 使用官方Python运行时作为基础镜像
FROM python:3.11-slim

# 设置工作目录
WORKDIR /app

# 复制依赖文件
COPY requirements.txt .

# 安装依赖
RUN pip install --no-cache-dir -r requirements.txt

# 复制应用代码
COPY . .

# 暴露端口
EXPOSE 8000

# 健康检查
HEALTHCHECK --interval=30s --timeout=10s --start-period=5s --retries=3 \
  CMD python -c "import urllib.request; urllib.request.urlopen('http://localhost:8000/health')"

# 启动应用
CMD ["python", "app.py"]
```

### 多阶段构建（优化镜像大小）
```dockerfile
# 构建阶段
FROM golang:1.21 AS builder
WORKDIR /app
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -o myapp .

# 运行阶段
FROM alpine:latest
COPY --from=builder /app/myapp .
EXPOSE 8080
CMD ["./myapp"]
```

## 常见Docker命令

```bash
# 构建镜像
docker build -t myapp:1.0 .

# 运行容器
docker run -d -p 8000:8000 --name myapp myapp:1.0

# 查看运行中的容器
docker ps

# 查看镜像
docker images

# 查看日志
docker logs myapp

# 进入容器
docker exec -it myapp bash

# 停止容器
docker stop myapp

# 删除容器
docker rm myapp

# 推送到仓库
docker tag myapp:1.0 username/myapp:1.0
docker push username/myapp:1.0
```

## 最佳实践

1. **使用官方镜像** - 从可信来源获取基础镜像
2. **多阶段构建** - 减少最终镜像大小
3. **非根用户** - 提升安全性
4. **最小化层数** - 合并命令减少镜像层
5. **使用.dockerignore** - 排除不需要的文件
6. **健康检查** - 监控容器状态
7. **固定版本** - 不要使用latest标签用于生产

## 安全最佳实践

```dockerfile
# ✓ 好的实践
FROM python:3.11-slim
RUN groupadd -r myuser && useradd -r -g myuser myuser
RUN pip install --no-cache-dir -r requirements.txt
USER myuser
COPY --chown=myuser:myuser . .

# ✗ 避免
FROM python:latest
RUN pip install -r requirements.txt  # 大量缓存
USER root
```

## 常见问题

**Q: 为什么容器会退出？**
A: 检查启动命令是否正确，查看日志了解错误。

**Q: 如何保持容器运行？**
A: 使用docker run -d 后台运行，或在CMD中使用长期运行的进程。

**Q: 如何优化镜像大小？**
A: 使用多阶段构建、alpine基础镜像、清理临时文件。
