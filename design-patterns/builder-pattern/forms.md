### 第一步: 确定 Builder 结构

**方案 A: 静态内部类**
```java
public class MyClass {
    public static class Builder {
        // Builder 代码
    }
}

使用: new MyClass.Builder().method().build();
```

**方案 B: 独立 Builder 类**
```java
public class MyClass { }
public class MyClassBuilder { }

使用: new MyClassBuilder().method().build();
```

**方案 C: Fluent Builder**
```java
public class FluentBuilder {
    public FluentBuilder method() { return this; }
}

使用: new FluentBuilder().m1().m2().m3().build();
```

## ⚠️ 常见陷阱检查

在实现前检查这些陷阱:

| 陷阱 | 风险 | 检查 | 防护 |
|------|------|------|------|
| 缺少参数验证 | 对象创建后非法 | ☐ | build() 中验证 |
| 参数易于推导 | 冗余的 Builder | ☐ | 减少参数或用工厂 |
| Builder 可变 | 多次 build() 结果不同 | ☐ | build() 后清空或重置 |
| 忘记防御副本 | 修改入参污染对象 | ☐ | 创建副本 |
| 内存泄漏 | Builder 占用资源未释放 | ☐ | 及时清空状态 |
