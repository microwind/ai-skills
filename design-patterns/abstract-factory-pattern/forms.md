# Abstract Factory Pattern - 诊断与规划指南

## 第1步: 需求诊断 - 是否需要Abstract Factory?

### 🔍 产品族识别矩阵

```
维度\需求    多个族  多产品  保证一致  灵活切换  创建复杂  
──────────────────────────────────────────────────────
UI系统        ✓     ✓      ✓        ✓      ✓    → USE AF
DB方言       ✓     ✓      ✓        ✓      ✓    → USE AF
游戏引擎      ✓     ✓      ✓        ✓      ✓    → USE AF
单对象       ✗     ✗      -        -      ✓    → 工厂方法
```

### 快速诊断清单 (勾选4项表示推荐)

- [ ] **多维产品族**: 系统有2个以上独立完整的产品族
- [ ] **族内产品**: 每个族有3个以上相关产品
- [ ] **一致性保证**: 必须确保同族产品配套使用
- [ ] **族切换频繁**: 运行时需灵活切换族
- [ ] **创建复杂**: 创建逻辑复杂或分散多处
- [ ] **扩展需求**: 经常添加新产品族

**诊断结果**: [勾选___项]
- ✅ 5-6项 → **强烈推荐** Abstract Factory
- ⚠️ 3-4项 → **可以使用** Abstract Factory  
- ❌ ≤2项 → **可能过度设计**

## 第2步: 创建族选择决策树

```
系统需多个产品族?
├─ 否 → 使用 Factory Method
└─ 是 → 需保证族内一致性?
   ├─ 否 → 使用独立工厂
   └─ 是 → 需灵活切换?
      ├─ 否 → Simple Abstract Factory
      └─ 是 → 需动态注册?
         ├─ 否 → 需配置化?
         │   ├─ 是 → Configuration-Driven AF
         │   └─ 否 → Singleton-Combined AF
         └─ 是 → Dynamic Registry AF (插件化)
```

### 实现方法对比

| 方法 | 复杂度 | 灵活性 | 场景 | 扩展成本 |
|------|--------|--------|------|---------|
| Simple | ⭐ | ⭐⭐ | 族固定 | 高 |
| Configuration-Driven | ⭐⭐ | ⭐⭐⭐ | 配置切换 | 低 |
| Singleton | ⭐⭐ | ⭐⭐ | 全局单例 | 中 |
| Dynamic Registry | ⭐⭐⭐ | ⭐⭐⭐ | 插件架构 | 低 |

## 常见陷阱与预防

### ⚠️ 陷阱1: 产品族混杂 **[最危险]**
```
❌ 错误: WindowsFactory创建Windows按钮+Mac文本框混合
✅ 修复: 在工厂返回前验证产品一致性
        assert button.getTheme().equals(field.getTheme())
```

### ⚠️ 陷阱2: 过度设计
```
❌ 错误: 仅1-2个产品也用Abstract Factory
✅ 修复: <3个产品→简单工厂; ≥3个→AF
```

### ⚠️ 陷阱3: 工厂爆炸
```
❌ 错误: 每个产品新增工厂类, 类数暴增
✅ 修复: Configuration-Driven 或 Dynamic Registry
```

### ⚠️ 陷阱4: 单例工厂测试困难
```
❌ 错误: SingletonFactory.getInstance() 硬编码
✅ 修复: 通过构造函数注入工厂, 便于Mock
```

### ⚠️ 陷阱5: 注册表内存泄漏
```
❌ 错误: Dynamic Registry 不支持注销
✅ 修复: 实现 unregister(id) 和 clear() 方法
```

## 快速参考

### 最小可用实现 (MVP)

```java
// 1. 定义接口
public interface Factory { ProductA createA(); }
public interface ProductA { void use(); }

// 2. 具体实现
public class FamilyAFactory implements Factory {
    public ProductA createA() { return new FamilyAProduct(); }
}

// 3. 使用
Factory factory = new FamilyAFactory();
factory.createA().use();
```

### 配置切换启动

```java
UIFactory factory = switch(config.getTheme()) {
    case "WINDOWS" -> new WindowsUIFactory();
    case "MAC" -> new MacUIFactory();
    default -> throw new IllegalArgumentException();
};
```

### 性能基准

| 操作 | 耗时 |
|------|------|
| 创建工厂 | < 0.01ms |
| 创建产品对 | < 0.01ms |
| 100万对象 | < 10s |

### 故障排查

| 现象 | 原因 | 解决 |
|------|------|------|
| ClassNotFoundException | 反射类名错误 | 检查配置/CLASSPATH |
| ClassCastException | 族混杂 | 验证工厂一致性 |
| OutOfMemoryError | 注册表未清理 | 实现清理机制 |
| 性能下降 | 频繁反射 | 缓存工厂或预加载 |
| ConcurrentModificationException | 并发 | 用ConcurrentHashMap |
