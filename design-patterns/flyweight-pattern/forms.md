## 第4步: 缓存策略规划

### 🔄 缓存层次选择

```
需要多少对象?
├─ 100-1K → 简单HashMap缓存
├─ 1K-10K → 分段缓存+LRU清理
├─ 10K+ → 多层缓存+内存限额
└─ 动态? → 自适应缓存+监控告警
```

**缓存策略对比**:
| 策略 | 命中率 | 内存占用 | GC负担 |易维护|
|------|-------|---------|-------|---|
| **无缓存** | 0% | 基准 | 高 | ⭐⭐⭐ |
| **全缓存** | 95%+ | 高 | 低 | ⭐⭐ |
| **LRU缓存** | 80-90% | 中 | 低 | ⭐⭐⭐ |
| **弱引用** | 70-85% | 中 | 很低 | ⭐ |

## 第5步: 实现规划 - 6步快速上手

**Step 1: 识别内在/外在状态** (20分钟)
```
□ 列出所有对象属性
□ 标记哪些是"内在"（所有实例相同）
□ 标记哪些是"外在"（每个实例不同）
□ 例: 字体(内在) vs 位置(外在)
```

**Step 2: 设计Flyweight对象** (20分钟)
```
□ 创建Flyweight类（只包含内在状态）
□ 设为immutable（不可变）
□ 覆盖equals()和hashCode()以支持缓存
□ 验证: 两个相同内在状态的对象可以共享
```

**Step 3: 实现FlyweightFactory** (20分钟)
```
□ 设计缓存数据结构（HashMap/WeakHashMap）
□ 实现getInstance()方法进行缓存查找
□ 定义缓存淘汰策略（LRU/WeakRef/时间戳）
□ 添加统计/监控功能（缓存命中率）
```

**Step 4: 处理外在状态** (20分钟)
```
□ 决定：外在状态由客户端传入还是Context管理？
□ 实现联合模式（Flyweight+Context）
□ 验证: 外在状态不存储在Flyweight中
□ 测试: 修改外在状态不影响其他引用
```

**Step 5: 性能优化** (20分钟)
```
□ 测量: 缓存命中率 (目标>80%)
□ 测量: 内存节省比例 (目标>50%)
□ 监控: GC暂停时间是否改善
□ 优化: 调整缓存大小/策略
```

**Step 6: 文档与监控** (15分钟)

□ 提供使用示例代码

## 第8步: 常见陷阱与预防

### ⚠️ 陷阱1: 内在状态定义不当

**问题**: 把应该是外在的状态放入Flyweight

**预防**:
- ✅ 严格区分内在（不变）vs 外在（变化）
- ✅ 定期审查内在状态定义

### ⚠️ 陷阱2: 缓存淘汰策略不当

**问题**: 内存泄漏或缓存击中率低

**预防**:
- ✅ 使用 WeakHashMap 或 LRUCache
- ✅ 设置最大容量限制

### ⚠️ 陷阱3: 忽视并发安全

**问题**: 多线程环境下数据竞争

**预防**:
- ✅ 使用 synchronized 或 ConcurrentHashMap
- ✅ 原子操作：查找+创建

### ⚠️ 陷阱4: 修改Flyweight对象

**问题**: Flyweight应该immutable

**预防**:
- ✅ 设为 final 类，无 setter 方法
- ✅ 所有字段都是 private final

## 快速参考与启动模板

### 🚀 30秒Flyweight模板

```java
// 1. Flyweight类
public class Flyweight {
    private final String intrinsicData;
    
    public Flyweight(String data) {
        this.intrinsicData = data;
    }
}

// 2. Factory
public class FlyweightFactory {
    private static Map<String, Flyweight> cache = new HashMap<>();
    
    public static Flyweight getFlyweight(String key) {
        if (!cache.containsKey(key)) {
            cache.put(key, new Flyweight(key));
        }
        return cache.get(key);
    }
}

// 3. 使用
Flyweight f1 = FlyweightFactory.getFlyweight("A");
Flyweight f2 = FlyweightFactory.getFlyweight("A");
assert f1 == f2; // 同一对象！
```
