### 第二步：选择实现方式

基于上面的矩阵，我选择这种实现方式:

#### ◯ 枚举（推荐用于 Java）
- ✅ **优点**: 最安全，防反射
- ❌ **缺点**: 启动时创建
- 📌 **适用**: Java 应用，需要最高安全性

```java
public enum [类名] {
    INSTANCE;
    
    // 初始化代码（可选）
    public void init() {
        // 初始化逻辑
    }
}
```

#### ◯ Bill Pugh（推荐用于 Java）
- ✅ **优点**: 延迟初始化，性能最佳
- ✅ **适用**: 初始化成本高的情况

```java
public class [类名] {
    private [类名]() {}
    
    private static class Holder {
        static final [类名] INSTANCE = new [类名]();
    }
    
    public static [类名] getInstance() {
        return Holder.INSTANCE;
    }
}
```

#### ◯ 饿汉式
- ✅ **优点**: 简单，性能最佳
- ❌ **缺点**: 启动时创建

```java
public class [类名] {
    private static final [类名] INSTANCE = new [类名]();
    
    private [类名]() {}
    
    public static [类名] getInstance() {
        return INSTANCE;
    }
}
```

#### ◯ 双检查锁（不推荐，复杂）
- 仅在必要时使用

#### ◯ Python 装饰器
```python
def singleton(cls):
    instances = {}
    def wrapper(*args, **kwargs):
        if cls not in instances:
            instances[cls] = cls(*args, **kwargs)
        return instances[cls]
    return wrapper

@singleton
class [类名]:
    pass
```

#### ◯ TypeScript 静态属性
```typescript
class [类名] {
    private static instance: [类名];
    
    public static getInstance(): [类名] {
        if (!this.instance) {
            this.instance = new [类名]();
        }
        return this.instance;
    }
}
```

## ⚠️ 常见陷阱检查

在实现前检查这些陷阱:

| 陷阱 | 风险 | 检查 |
|------|------|------|
| 忘记 `volatile` | 多线程失败 | ☐ |
| 没有 `readResolve()` | 反序列化失败 | ☐ |
| 没有防反射 | 可被反射攻击 | ☐ |
| 全局状态污染 | 难以测试 | ☐ |
| 初始化异常未处理 | 应用启动失败 | ☐ |
